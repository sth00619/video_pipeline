# YouTube API 키 발급 안내(비개발자용)

이 문서는 결제 담당자가 브라우저에서 키를 발급하고 운영 담당자에게 안전하게 전달하기 위한 절차입니다. 키 값은 관리자 화면에 표시하지 않으며, 서버 환경변수에만 저장합니다.

## 1. Google Cloud 프로젝트 만들기

1. [Google Cloud Console](https://console.cloud.google.com/)에 회사 Google 계정으로 로그인합니다.
2. 상단 프로젝트 선택 → **새 프로젝트**를 누릅니다.
3. 프로젝트 이름을 `video-pipeline`처럼 입력하고 **만들기**를 누릅니다.

## 2. YouTube Data API v3 켜기

1. 왼쪽 메뉴 **API 및 서비스 → 라이브러리**로 이동합니다.
2. `YouTube Data API v3`를 검색합니다.
3. **사용**을 누릅니다.

## 3. API 키 만들기

1. **API 및 서비스 → 사용자 인증 정보**로 이동합니다.
2. **사용자 인증 정보 만들기 → API 키**를 누릅니다.
3. 생성된 키를 복사합니다.
4. 키 화면에서 **API 제한**을 `YouTube Data API v3`로 제한합니다.
5. 서버가 고정 IP를 사용하는 경우 **애플리케이션 제한 → IP 주소**도 설정합니다. 고정 IP가 없으면 우선 API 제한만 적용하고, 운영 배포 후 IP 제한을 추가합니다.

## 4. 서버에 전달하기

운영 담당자에게 메신저로 평문 전달하지 말고 회사 비밀번호 관리 도구를 사용합니다. 운영 담당자는 프로젝트 루트 `.env`에 다음 한 줄을 넣습니다.

```env
YOUTUBE_API_KEY=발급받은_키
```

그 후 서버에서 다음 명령으로 재시작합니다.

```bash
docker compose up -d --build fastapi-workers spring-app
```

## 5. 연결 확인

관리자 화면의 **외부 API 연결 상태**에서 `YouTube Data API v3 · 연결됨`을 확인합니다. 화면에는 키 값 자체가 절대 표시되지 않고 `연결됨/키 없음` 상태만 표시됩니다.

키가 없거나 호출이 실패하면 실제 조회수·구독자 수를 추정하지 않고 `unavailable`로 표시합니다.

## 비용/보안 메모

- YouTube Data API는 요청마다 quota가 차감되므로 키워드 검색을 무제한 반복하지 않습니다. 서버에서 1시간 캐시를 사용합니다.
- 결제 계정 연결은 Google Cloud Console의 **결제** 메뉴에서 진행합니다. 결제수단 등록 자체와 API 사용량 과금은 별도이며, 사용량·예산 알림을 함께 설정하는 것을 권장합니다.
- 키가 노출되었으면 즉시 사용자 인증 정보 화면에서 키를 삭제하고 새 키를 발급합니다.
